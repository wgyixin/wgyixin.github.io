gitlab_and_github

Posted on 2017-09-17

使用工作电脑时，难免会遇到公司gitlab账号和个人github账号并存的问题，此时就需要区分目录，去提交不同的host。因此我们需要多份SSH-KEY对应不同的环境。主要的步骤如下（附才得坑）：

生成公司gitlab key

“youremail@yourcompany.com” 为你公司域账号邮箱。
在~/.ssh/目录会生成id_rsa 和 id_rsa.pub私钥和公钥。 我们将id_rsa.pub中的内容粘帖到公司gitlab服务器的SSH-key的配置中.

生成个人github key

“youremail@yourgit.com” 为你github中注册的邮箱。
在~/.ssh/目录会生成github_rsa 和 github_rsa.pub私钥和公钥。 我们将github_rsa.pub中的内容粘帖到github的SSH-key的配置中.

验证是否连接成功

如果输出 success，表示配置成功。
如果显示 The authenticity of host 'github.com (192.30.252.129)' can't be established. 表示配置缺失，此时还需要执行一些操作配置才行：

  • 查看 ~/.ssh 下的 known_hosts 文件，会发现 其中并没有 github.com 的标识，因此我们需要主动将host写入改文件
  • 执行 ssh-keyscan -t rsa github.com >> ~/.ssh/known_hosts 会将github host追加到文件末尾
  • 再进行 ssh -T 链接 表示链接成功。完成key 配置。

提交project 到 github

在 本地project 目录下 执行

  • git init 初始化
  • git config user.name yourgithubname and git config user.email yourgithubemail 配置该目录下使用的git账号，注意 不要轻易使用 git config --global user.name这样是会修改全局配置。
  • github中创建同名project 在本地目录下执行 git remote add origin git@yourproject.git
  • git add ; git commit ; git push 即可以提交了
